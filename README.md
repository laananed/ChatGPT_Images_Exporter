# ChatGPT Image Downloader

一个本地使用的 Chrome / Edge Manifest V3 扩展，用来从你自己的 ChatGPT 图片页面或包含生成图片的会话页面中批量扫描、解析并下载图片。

当前版本：`0.1.0`

## 功能特性

- 支持 `chatgpt.com` 和旧域名 `chat.openai.com`。
- 自动向下滚动页面，收集当前页面能够加载到的图片卡片。
- 会尝试打开图片详情页或预览层，优先解析大图/原图地址，而不是直接保存缩略图。
- 会从页面资源中补充可下载图片候选，并合并到稳定的 `scanList` 中。
- 带有原图质量检查，尽量避免把低分辨率缩略图、页面素材、图标或模板资源误保存为结果图。
- 支持下载文件夹、账号/标签、滚动次数、速度档位、下载模式和时间范围过滤。
- 支持“只下载新增”和“下载全部”两种模式；新增模式会基于本地下载历史跳过已下载图片。
- 支持按“今天”或自定义开始/结束时间过滤，并可选择跳过或包含无法判断时间的图片。
- 弹窗内显示任务进度、统计指标和诊断日志。
- 支持导出 `jobReport` 为 CSV、复制失败清单、复制日志、清空日志，以及重试解析失败/下载失败项。
- 使用浏览器自带的 `downloads` API 下载，不接管账号，不保存账号密码，不把图片上传到第三方服务。

## 安装方式

1. 下载或 clone 本仓库到本地。
2. 打开 Chrome 或 Edge 的扩展管理页：
   - Chrome：`chrome://extensions`
   - Edge：`edge://extensions`
3. 打开“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本项目根目录，也就是包含 `manifest.json` 的文件夹。
6. 如果安装前已经打开了 ChatGPT 页面，请刷新该页面，让内容脚本重新注入。

修改代码后，需要回到扩展管理页点击“重新加载”，然后刷新 ChatGPT 标签页。

## 使用方式

1. 打开 `https://chatgpt.com/`，进入 ChatGPT Images 页面，或进入包含生成图片的会话页面。
2. 点击浏览器工具栏里的扩展图标。
3. 按需设置：
   - 下载文件夹：默认 `ChatGPT Images`。
   - 账号/标签：可选，默认会使用 `default`，便于多账号或多用途分类。
   - 滚动次数：控制扩展向下加载图片的次数。
   - 速度档位：稳定、标准、快速，对应不同下载并发和间隔。
   - 下载模式：只下载新增或下载全部。
   - 时间范围过滤：不过滤、今天、自定义开始/结束时间。
   - 未知时间处理：跳过未知时间或包含未知时间。
4. 点击“扫描并下载”。
5. 任务运行时尽量保持当前 ChatGPT 标签页打开，不要频繁切换页面状态。

下载完成后，可以在弹窗底部查看统计、导出 CSV、复制失败清单或重试失败项。

## 下载目录与文件名

文件会保存到浏览器默认下载目录下，路径结构大致如下：

```text
ChatGPT Images/default/2026-05-23/20260523-153012-a1b2c3/0001-2026-05-23-9f8e7d6c-prompt-title.png
```

目录含义：

- `ChatGPT Images`：弹窗里的下载文件夹。
- `default`：账号/标签。
- `2026-05-23`：任务日期。
- `20260523-153012-a1b2c3`：单次任务的运行 ID。
- `0001-日期-短哈希-prompt-title.ext`：单张图片文件名。

文件名会自动清理系统不允许的字符，并尽量保留图片序号、日期、短哈希和 prompt 摘要，方便之后核对。

## 时间过滤说明

扩展会从多个位置尝试提取图片时间：

- 图片卡片附近的 `time` 元素。
- 常见日期属性，例如 `datetime`、`title`、`aria-label`、`data-created-at`。
- 图片详情或上下文文本中的日期。
- 下载准备阶段拿到的响应头，优先使用 `last-modified` 作为补充。

因为 ChatGPT 页面结构和图片来源可能变化，部分图片可能无法可靠判断时间。使用时间过滤时，可以选择跳过这些未知时间图片，或把它们包含进下载任务。

## 诊断与导出

弹窗底部会保留最近的诊断日志和任务报告。排查时通常最有用的是：

- 最终状态里的扫描数量、解析成功数量、去重数量、质量失败数量、下载提交数量和下载失败数量。
- “复制日志”得到的完整日志。
- “复制失败清单”得到的 JSON。
- “导出 CSV”得到的 `jobReport` 清单。

如果只看到 `/images-app/*.webp` 之类的资源，通常表示扩展只扫到了 ChatGPT 页面自身的 UI 或模板素材，还没有扫到你的生成图片。可以确认页面已经登录、图片已经可见，再刷新页面重试。

## 权限说明

`manifest.json` 中声明的主要权限：

- `activeTab`：读取当前激活的 ChatGPT 标签页。
- `downloads`：通过浏览器下载管理器提交图片下载。
- `scripting`：在需要时确保内容脚本可用。
- `storage`：保存设置、任务状态、诊断日志、下载历史和 `jobReport`。

主机权限覆盖 ChatGPT 页面及 OpenAI 图片/CDN 相关域名，用于识别、校验和下载生成图片资源。

## 项目结构

```text
manifest.json
src/
  background.js   后台任务、下载队列、历史去重、报告与日志
  content.js      页面扫描、详情页解析、原图候选与质量检查
  popup.html      扩展弹窗结构
  popup.css       弹窗样式
  popup.js        弹窗交互、设置、状态刷新、CSV 导出和失败重试
```

本项目目前不需要构建步骤，加载根目录即可作为未打包扩展运行。

## 已知限制

- 扩展只能扫描当前页面已经加载或通过滚动能够加载出来的内容；ChatGPT 没有放进 DOM 的旧图片无法直接读取。
- 原图解析依赖 ChatGPT 当前页面结构和图片资源格式；如果页面结构变化，可能需要更新解析逻辑。
- 任务会逐张打开详情或预览层来提高原图命中率，因此比单纯保存缩略图更慢。
- 当前版本继续逐张提交浏览器下载任务，不会强制打包 ZIP，也不会为了 ZIP 把所有大图一次性加载进内存。
- “重试失败项”默认重试解析失败和下载失败；质量检查失败项不会默认重试。
