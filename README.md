# ChatGPT Image Downloader

一个本地使用的 Chrome / Edge 扩展原型，用来在你自己的 ChatGPT 图片页面里批量下载当前能够扫描到的生成图片。

## 当前能力

- 只在 `chatgpt.com` 和旧域名 `chat.openai.com` 生效。
- 自动向下滚动，边加载边收集图片缩略图。
- 逐张打开图片详情页或预览层，尝试提取详情里的大图/原图地址。
- 支持滚动次数、下载间隔和下载文件夹。
- 弹窗内提供诊断日志，可以复制或清空，用于排查原图解析失败原因。
- 使用浏览器自己的 `downloads` API 下载，不接管你的账号，不保存账号密码。

## 安装方式

1. 打开 Chrome 或 Edge。
2. 进入扩展管理页：
   - Chrome: `chrome://extensions`
   - Edge: `edge://extensions`
3. 打开“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本项目文件夹。

## 使用方式

1. 打开 `https://chatgpt.com/`，进入你的图片页面或包含图片的会话页面。
2. 点击浏览器工具栏里的扩展图标。
3. 按需调整下载文件夹、滚动次数和下载间隔。
4. 点击“扫描并下载”。

下载文件会放到浏览器默认下载目录下的 `ChatGPT Images` 文件夹中，除非你在弹窗里改了文件夹名。

如果解析结果不符合预期，可以展开查看弹窗底部的“诊断日志”，点击“复制”后把日志内容发给排查对话。

## 如果仍然失败，优先提供这些信息

排查时最有用的是：

- 当前页面地址路径，例如 `https://chatgpt.com/images/` 或某个会话页，不需要提供账号信息。
- 弹窗最终状态里的：扫描张数、解析原图张数、详情失败张数、下载失败数。
- 诊断日志里从“任务启动”到“内容脚本扫描结束”的完整文本，尤其是 `缩略图收集完成`、`已从页面资源中直接发现可下载候选`、`原图解析失败` 这几类记录。
- 浏览器下载列表中是否出现任务；如果出现但失败，提供失败原因文字。
- 任选一张你认为应该被下载的图片截图，最好包含图片周围的按钮或标题区域。

如果日志里只看到 `/images-app/*.webp`，通常表示扩展只扫到了 ChatGPT Images 页面自己的示例/模板素材，还没有扫到你的生成图。

## 重要限制

这是第一版可用原型，核心限制是：它只能扫描当前页面能够加载出来的内容。ChatGPT 页面如果没有把某些旧图片加载进 DOM，扩展就看不到那些图片。

为了尽量拿到原图，扩展会自动逐张打开图片详情再关闭。这个过程会比直接下载缩略图慢，也依赖 ChatGPT 当前页面结构。如果某张图片详情打不开，或详情里没有可解析的大图地址，它会被统计为失败并跳过。

## 建议试跑方式

第一次试跑建议：

- 滚动次数设为 `5` 到 `10`。
- 先用少量图片验证文件大小和画面完整性。
- 下载成功后再把滚动次数调大。

如果出现“扫描失败”，刷新 ChatGPT 页面后再点扩展。刚安装扩展时，已经打开的 ChatGPT 标签页可能还没有注入脚本。
